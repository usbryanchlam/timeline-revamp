<objective>
Install React Router v7 and restructure the app from "single root component" to "router-driven route tree." Create the route shell that satisfies APP-03: `/` (public reel), `/u/:handle` (per-handle public reel), and `/app/*` (the tree that Phase 4 will wrap in Auth0).

Purpose: Phase 3 introduces multi-page navigation. Today `src/App.tsx:7` is a single line that picks `<Reel />` vs `<ReducedMotionReel />`. After this plan, that decision moves down into route components, and the app has the hierarchical structure the rest of the v1 needs (bottom nav, auth wrapper, per-handle public pages).

Critical forward-compat: `AppLayout` must be a real layout component wrapping `<Outlet />`, not inlined logic. Phase 4 will drop `<Auth0Provider>` + redirect-to-login around it without touching public routes. If you collapse `AppLayout` into App.tsx or the route config, Phase 4 has to refactor.

Note on reduced-motion duplication: `PublicReelRoute` and `AppReelRoute` will both call `usePrefersReducedMotion()` and branch independently. This is acceptable for v1 — they are short, identical 3-line bodies. Future refactor candidate (Phase 9): extract `<ReelView />` shared between public/private route trees once user-data wiring lands. Don't extract early; the shared abstraction only earns its keep when each side has different data inputs.

Output: React Router v7 installed, `App.tsx` returns a `RouterProvider`, six route components in `src/routes/`, dev + build still green, existing reel still reachable at `/`.
</objective>
