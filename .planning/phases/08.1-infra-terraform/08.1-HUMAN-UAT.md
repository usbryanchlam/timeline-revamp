### 2. CORS on photos bucket via `null_resource.photos_cors`
expected: `aws s3api put-bucket-cors` against OCI S3-compat endpoint applies the rules from `var.photos_cors_rules`; `aws s3api get-bucket-cors` returns them
result: **not applicable for this stack → resource removed; no operator action required**
context: Modern OCI Object Storage exposes **no bucket-level CORS configuration at any layer** — not the Console UI (no CORS tab in Edit Bucket), not the Native API (no `corsRules` field on the bucket data model), not the S3-compat API (PutBucketCors returns `NotImplemented`). The 2026-05-27 attempt was based on an incorrect memory note that has now been rewritten (`feedback_oci_cors_via_s3.md` v3 — supersedes both the prior "S3-compat works" and "Console UI works" claims). **This stack does not need bucket CORS** because: (a) browser uploads go through PARs minted by `server/oci/parClient.ts`, and PAR responses carry their own `Access-Control-Allow-Origin` header automatically; (b) photo display in `PhotoGrid.tsx` / `PhotoViewer.tsx` uses bare `<img src={thumbUrl}>` with no `crossOrigin` attribute, which browsers do not subject to CORS enforcement. If a future feature adds canvas pixel reads or credentialed fetch on bucket URLs, the workaround is to front the bucket with an OCI API Gateway — there is no direct path. Resource removed from `infra/terraform/storage.tf` (commit `1dbd54c`). Variable `var.photos_cors_rules` preserved in `variables.tf` as documentation of intent only.

### Gap 1 — Phase 6 photo upload CORS (closed: not applicable)
**What was missing:** Original concern was that cross-origin browser-to-bucket uploads from `https://timeline.bryanlam.dev` would fail preflight without bucket CORS.
**Resolution (2026-05-28):** Closed as **not applicable** after verification of the actual code paths. Browser uploads in this stack do not hit the bucket directly — they hit a one-time **Pre-Authenticated Request (PAR) URL** minted by `server/oci/parClient.ts`, and PAR responses include `Access-Control-Allow-Origin` automatically. Photo display uses bare `<img src={thumbUrl}>` without `crossOrigin`, which browsers do not subject to CORS enforcement. No operator action required for v1.
**Re-open trigger:** If a future feature introduces canvas pixel reads, `getImageData`, or credentialed `fetch()` against bucket URLs, re-evaluate.
