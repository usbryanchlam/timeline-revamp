# Phase 08.1 — Validation Strategy

> Per-phase validation contract for feedback sampling during execution.

This phase has no traditional unit-test framework. Validation comes from declarative-IaC tools: `terraform validate`, `terraform plan`, `actionlint`, `cloud-init schema`, plus a one-shot integration test that runs a real `terraform apply` against the OCI tenancy.

---

## Test Infrastructure

| Property | Value |
|----------|-------|
| **Framework** | `terraform validate` + `terraform plan` + `actionlint` + `cloud-init schema` + manual `terraform apply` against live OCI tenancy |
| **Config file** | `infra/terraform/versions.tf`, `.github/workflows/terraform.yml`, `infra/cloud-init.yaml` |
| **Quick run command** | `cd infra/terraform && terraform fmt -check && terraform validate` |
| **Full suite command** | `cd infra/terraform && terraform fmt -check && terraform validate && terraform plan -detailed-exitcode` plus `actionlint .github/workflows/terraform.yml` plus `cloud-init schema --config-file infra/cloud-init.yaml` |
| **Estimated runtime** | ~2s quick / ~30s full / ~5-20min real apply |

---

## Sampling Rate

- **After every task commit:** Run `terraform fmt -check && terraform validate` (~2s)
- **After every plan wave:** Run full quick suite + `actionlint .github/workflows/terraform.yml` + `cloud-init schema --config-file infra/cloud-init.yaml` (~30s)
- **Before `/gsd-verify-work`:** Full suite green AND one successful real `terraform plan -detailed-exitcode` against the live OCI tenancy
- **Max feedback latency:** 30 seconds for the wave-level suite; the real-apply test runs once per phase gate

## Wave 0 Requirements

Wave 0 is the bootstrap that cannot be bootstrapped by Terraform itself. Plans MUST order these before Wave 1's first `terraform init`:

- [ ] `infra/terraform/versions.tf` — pins `required_version = "~> 1.10.0"` (NOT `>= 1.10` — Terraform 1.11.2 is broken with OCI S3-compat per `hashicorp/terraform#36742`) and `oracle/oci ~> 6.0`
- [ ] State bucket created via OCI CLI (`oci os ns get`, `oci os bucket create --name timeline-tfstate --versioning Enabled`, `oci os bucket update --retention-rules` for 90-day delete-retention)
- [ ] Customer Secret Key generated for OCI S3-compat auth: `oci iam customer-secret-key create --user-id <user-ocid>`
- [ ] GHA Secrets populated: `OCI_S3_ACCESS_KEY`, `OCI_S3_SECRET_KEY` (one-time, used ONLY by the s3 backend; everything else uses OIDC)
- [ ] GHA Variables populated: `OCI_DOMAIN_URL`, `OCI_REGION`, `OCI_TENANCY_OCID`, `OCI_COMPARTMENT_OCID`
- [ ] GitHub Environment `production` created with user as required reviewer (manual GitHub UI step; cannot be TF-managed)
- [ ] SSH public key file present at the path declared in `terraform.tfvars` (`~/.ssh/oci-timeline.pub`)
- [ ] Existing Phase 6 photos bucket either deleted (operator confirms test-data-only) OR `terraform import oci_objectstorage_bucket.photos n/<namespace>/b/timeline-photos` before first apply
- [ ] (Optional) `tflint-ruleset-oci` installed if planner adopts tflint
- [ ] (Optional) `actionlint` installed locally for per-commit GHA workflow validation
