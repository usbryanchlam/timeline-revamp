## Affected Pages/Routes

- **`/`** — landing reel (seeded `demo` user). Test that motion lands within 5s, photos render, no auth flash, `prefers-reduced-motion` fallback renders identical content as a static list.
- **`/login`** — Auth0 Universal Login redirect. Test signup + signin paths for both Google social and email/password connections.
- **`/app`** (private) — bottom-nav shell with Reel + Trips views. Test redirect to /login when unauthenticated, redirect to handle picker when handle is null.
- **`/app/cities/new`** — add-city sheet with map picker. Test reverse-geocode happy path, no-result state, photo upload happy + retry paths.
- **`/u/:handle`** — public reel. Test handle lookup, 404 for nonexistent, 0-city empty state, 1-city orbit camera, 2+ cities flyover, reduced-motion fallback.
- **`/api/public/reels/:handle`** — public unauthenticated GET. Verify Nginx 60s cache headers, payload includes absolute photo URLs.
- **`/api/cities/reorder`** — bulk PATCH transactional swap. Verify deferred-unique constraint behavior with intermediate collisions.
- **`/api/cities/:id/photos/upload-url`** — PAR mint. Verify 10-min TTL, scoped to one object key.

## Key Interactions to Verify

- **Reel gestures (mobile, on real device or BrowserStack):**
  - Long-press ≥200ms triggers SCRUBBING; horizontal drag scrubs time.
  - Vertical flick ≥30px in <300ms triggers CHAPTER_SWIPE.
  - Two-finger pinch enters MAP_INTERACT, releases to IDLE after 3s idle.
  - Single tap on photo opens detail sheet; tap on background toggles PAUSED.
  - Three-finger gesture is NOT intercepted (iOS accessibility passes through).
  - iOS back-edge-swipe (left 20px) is NOT intercepted.
  - `visibilitychange` to hidden enters SUSPENDED; visible returns to IDLE.
  - `orientationchange` enters SUSPENDED, returns to IDLE after 300ms with resized canvas.
  - Browser pull-to-refresh is disabled on the reel route (`overscroll-behavior: none`).

- **Add city flow:**
  - Click on map → reverse-geocode populates city name, country, country code, emoji.
  - DatePicker honors min (today − 10 years) and max (today).
  - Photo upload: HEIC → JPEG client-side conversion succeeds.
  - EXIF stripping: assert output has NO GPS coordinates.
  - Resize: assert output ≤ 2048px on longest edge, ≤ 5MB.
  - Thumbnail: assert 400px square crop generated.
  - Upload retry: simulate network failure on first PUT, verify 3 retries with backoff.

- **Reorder cities:**
  - Drag-and-drop emits one bulk PATCH with the full new ordering.
  - Backend wraps in `db.transaction()`; intermediate states never visible to other clients.
  - Optimistic UI updates immediately; rollback on 4xx response.

- **Auth flows:**
  - Signup via Google → handle picker → /app.
  - Signup via email/password → email verification → handle picker → /app.
  - Signin → /app, no handle picker re-prompt if handle exists.
  - Logout clears Auth0 state, redirects to /.
  - JWT expiry: silent refresh works on /app; on failure, redirect to /login.
  - Public routes (`/`, `/u/:handle`) never trigger Auth0 silent auth (no flash).

## Edge Cases

- **Public reel:**
  - Handle exists, user has 0 cities: world-view map + "No trips yet" caption.
  - Handle exists, user has 1 city: orbit camera (8s, zoom 14, pitch 60).
  - Handle exists, user has 2+ identical coordinates: collapsed chapter group.
  - Handle does not exist: 404 page with home link.
  - Handle uppercase in URL: redirect to lowercase canonical.

- **Photo upload:**
  - HEIC file from iPhone Photos: converted to JPEG client-side.
  - File >5MB after resize: rejected with clear error.
  - Network drops mid-upload: retry exponential backoff, then user-friendly "tap to retry."
  - 11th photo on a city: rejected with "10 photo limit" message (NOT a silent fail).

- **MP4 render:**
  - 6th render in 24h: 429 with `Retry-After` header showing seconds until reset.
  - Render fails (FFmpeg error): notification.kind='mp4_failed' with retry button.
  - User cancels mid-render: job moved to "cancelled," worker stops on next frame poll.
  - Tab closed during server render: render continues; notification appears next visit.

- **Handle picker:**
  - User types `admin`: rejected (reserved).
  - User types `bryan` and it's taken: shows "taken, try `bryan-2`."
  - User types `Bryan`: lowercased to `bryan` for comparison; preserves display preference if `display_name` differs.
  - User types `b`: rejected (min length 3).
  - User types `bryan!@#`: rejected (`[a-z0-9-]` only).

- **Map picker:**
  - User clicks ocean: reverse-geocode returns no city → "Click somewhere on land 😉" message.
  - User clicks far-edge of map: lat/lng wrap correctly (no -91° latitude bugs).

- **Database:**
  - Cascade delete: deleting a user removes all cities and photos in one transaction.
  - Cascade delete: deleting a city removes all photos in one transaction; orphan cleanup catches any Object Storage objects whose row is gone.
  - Reorder + concurrent write: optimistic locking via `updated_at` on cities, returns 409 if stale.

## Critical Paths (must work end-to-end before launch)

1. **30-second test:** unauth visitor opens `timeline.bryanlam.dev` on iOS Safari → reel motion within 5s → understands product within 15s.
2. **Signup → first city → public URL:** Auth0 signup → pick handle → add one city with one photo → visit `/u/myhandle` in incognito → see the city.
3. **MP4 export (whichever ladder rung shipped):** trigger export → receive MP4 (or GIF, or "record screen" hint) → file plays on iOS, Android, desktop browsers.
4. **Reduced-motion path:** OS-level reduce-motion enabled → public reel renders static chapter list → axe-core 0 violations.
5. **Deploy pipeline:** push tag `v0.x.x` → GitHub Actions builds → OCI Container Registry receives → VM auto-deploys → `/health` returns 200.
6. **TLS auto-renewal:** Let's Encrypt cert is 30 days from expiry → certbot renews → no downtime, no manual intervention.
