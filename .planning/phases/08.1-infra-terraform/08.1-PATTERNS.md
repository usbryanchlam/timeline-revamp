### `infra/terraform/backend.tf` (tf-stack, state → OCI S3-compat)

**Analog:** RESEARCH.md Pattern 1 (lines 186-209). No in-repo analog.

**Excerpt to copy:**
```hcl
terraform {
  backend "s3" {
    bucket = "timeline-tfstate"
    key    = "infra/terraform.tfstate"
    region = "<REGION>"          # operator fills via `terraform init -backend-config=...`

    endpoints = {
      s3 = "https://<NAMESPACE>.compat.objectstorage.<REGION>.oraclecloud.com"
    }

    use_lockfile                = true   # TF 1.10+ native S3 conditional-write locking
    use_path_style              = true   # OCI S3-compat requires path-style URLs
    skip_credentials_validation = true
    skip_metadata_api_check     = true
    skip_region_validation      = true
    skip_requesting_account_id  = true
    skip_s3_checksum            = true
  }
}
```

**Notes for executor:**
- D-09 bootstrap path: backend values supplied via `terraform init -backend-config="bucket=..." -backend-config="region=..." -backend-config="endpoints={s3=\"...\"}"`. Do NOT hardcode region/namespace inside `backend.tf` — the file is committed; the values are tenant-specific.
- Auth via env vars (`AWS_ACCESS_KEY_ID`, `AWS_SECRET_ACCESS_KEY`, plus `AWS_REQUEST_CHECKSUM_CALCULATION=when_required`) — set by GHA from `OCI_S3_ACCESS_KEY`/`OCI_S3_SECRET_KEY` secrets.
- **Do NOT** use deprecated `force_path_style = true`. Anti-Pattern (RESEARCH line 727).

```hcl
resource "null_resource" "photos_cors" {
  triggers = {
    cors_rules_hash = sha256(jsonencode(var.photos_cors_rules))
    bucket_id       = oci_objectstorage_bucket.photos.id
  }

  provisioner "local-exec" {
    command = <<-EOT
      export AWS_REQUEST_CHECKSUM_CALCULATION=when_required
      export AWS_RESPONSE_CHECKSUM_VALIDATION=when_required
      aws s3api put-bucket-cors \
        --endpoint-url "https://${data.oci_objectstorage_namespace.this.namespace}.compat.objectstorage.${var.region}.oraclecloud.com" \
        --bucket "${oci_objectstorage_bucket.photos.name}" \
        --cors-configuration '${jsonencode({ CORSRules = var.photos_cors_rules })}'
    EOT
  }

  depends_on = [oci_objectstorage_bucket.photos]
}
```

**Notes for executor:**

Trust RESEARCH (the freshest verification) — use the `null_resource` + `aws s3api` path. - **Anti-Pattern** (RESEARCH line 729): `depends_on` is non-negotiable.
- CORS values: extract from current Phase 6 bucket via `oci os bucket get-bucket-cors-policy` + `aws s3api get-bucket-cors --endpoint-url ...` at plan time (D-15). Default if absent: `<specifics>` line 178.

**Notes for executor:**
- The header comment `# Managed by Terraform — edit infra/cloud-init.yaml, not the rendered file` is **required** per `<specifics>` line 174.

- **Pitfall 3** (RESEARCH lines 786-790): `iptables-persistent` MUST be in the top-level `packages:` block, NOT in `runcmd:`. cloud-init's module order (packages → runcmd) guarantees the binary exists when runcmd fires. Don't merge them.
- The `git clone ... || true` in runcmd makes cloud-init idempotent (replays without erroring on existing clones — though cloud-init only runs once per Pitfall 4).
- Auth0 callback URL update (`setup.sh:71` final message item 5) is NOT in cloud-init — it's a manual Auth0 dashboard step. Belongs in DEPLOY.md, not the YAML.

## Shared Patterns

### Pattern: OCID variable comments
**Source:** RESEARCH Pitfall 7 (lines 810-814)
**Apply to:** `variables.tf`, `terraform.tfvars.example`

Every OCID variable gets a one-line comment showing the expected format prefix:
```hcl
variable "tenancy_ocid" {
  type        = string
  description = "Root tenancy OCID. Format: ocid1.tenancy.oc1..aaaa..."
}
variable "compartment_ocid" {
  type        = string
  description = "Target compartment OCID. Format: ocid1.compartment.oc1..aaaa... (NOT tenancy)"
}
```

---

### Pattern: `lifecycle { ignore_changes = [defined_tags] }`
**Source:** RESEARCH Pitfall 8 (lines 816-820)
**Apply to:** Every `oci_*` resource that accumulates `Oracle-Tags` defined-tags (effectively all of them).

```hcl
resource "oci_core_instance" "timeline" {
  # ... config ...
  lifecycle {
    ignore_changes = [defined_tags]
  }
}
```

Without this, every `terraform plan` shows spurious "defined_tags changed" diffs and the drift-detection cron raises false-positive Issues.

---

### Pattern: AWS-S3-compat env vars for OCI
**Source:** RESEARCH Pattern 1 + Pattern 4 + Pitfall 9
**Apply to:** `backend.tf` (implicitly via GHA env), `storage.tf` (explicitly via `local-exec`), `.github/workflows/terraform.yml` (top-level `env:` block)

```bash
AWS_REQUEST_CHECKSUM_CALCULATION=when_required
AWS_RESPONSE_CHECKSUM_VALIDATION=when_required
```

These two env vars are **mandatory** wherever AWS CLI v2.23.5+ talks to OCI S3-compat. Set globally in the GHA workflow `env:` block (RESEARCH line 542-543) so they cover backend init, plan, apply, AND the CORS `local-exec`.

---

### Pattern: `depends_on` for cross-resource ordering
**Source:** RESEARCH Anti-Patterns (line 729) + Pitfall 5 (lines 798-802)
**Apply to:** `storage.tf` (null_resource → bucket), `compute.tf` (data sources → instance)

Explicit `depends_on` is required wherever an implicit dependency might race:

- `data.oci_core_private_ips.primary` → `depends_on = [oci_core_instance.timeline]` (or chain off the vnic data source)
