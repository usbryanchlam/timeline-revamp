<objective>
Wire light + dark theme support driven by the OS `prefers-color-scheme` media query. Today the app hardcodes a dark palette in `:root`. After this plan, dark stays the default but light-mode users get the DESIGN.md light palette automatically — applied to **chrome only**. The reel canvas stays dark in both modes per DESIGN.md:72 ("The public reel is always dark"). No toggle in v1 — system preference is the entire UX.

Purpose: Satisfy APP-02 ("Tailwind theme supports light + dark, system preference by default"). Get the token wiring right so Phase 4+ component work can use OS-driven theming without redoing this.

**Decision: direct override of existing `--color-*` vars inside `@media (prefers-color-scheme: light)`. No new semantic-alias token layer.** Rationale: components already use Tailwind utilities (`bg-bg`, `text-ink`, `border-line`) that resolve via `tailwind.config.ts` directly to `--color-*` CSS vars (see `tailwind.config.ts` colors block). Adding a parallel `--bg`/`--surface`/`--text` alias layer would be dead code unless components migrate — and there's no reason to migrate. We just flip the underlying values inside a light media query. Zero new token names, zero migration cost, ~30 fewer lines than the alias approach.

**Decision: CSS-media-query approach, no `ThemeProvider` React context, no class-on-html.** Rationale: DESIGN.md does not request a manual toggle in v1; adding a context + persisted-preference layer is unused complexity. Pure-CSS theming has zero JS cost, zero hydration flicker. If a manual override is wanted later (v2), we add a context and switch Tailwind to `darkMode: ['class', '[data-theme="dark"]']` then.

**Pre-existing drift to fix in this plan:** `tailwind.config.ts:25` and `src/index.css:14` currently use `#F5B83A` for amber-500 and `#C28A1E` for amber-600. DESIGN.md:86-87 specifies `#FFD470` and `#E8B040`. The drift is small but matters: AA contrast claims about amber-600 on light backgrounds only hold for `#E8B040`, not the much-darker `#C28A1E`. We reconcile both files to DESIGN.md as part of this theme work, since the theme is the natural place to surface and resolve color drift.

Output: Light-mode chrome overrides in `index.css`, `darkMode: 'media'` in Tailwind config, amber tokens reconciled to DESIGN.md, a TODOS note for the deferred manual toggle.
</objective>
