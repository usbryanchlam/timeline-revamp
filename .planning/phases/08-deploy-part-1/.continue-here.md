## Critical Anti-Patterns

| Pattern | Severity | Why it matters |
|---|---|---|
| Trusting cloud-init "Cloud-init complete." final_message | medium | The final_message fires even when intermediate modules errored. Phase 8 Wave 3 saw `package_update_upgrade_install` fail, runcmd continue, and the final message still print. Verify by `sudo cloud-init status --long` AND tail of `/var/log/cloud-init-output.log` for errors, not just the closing banner. |
| `ports:` lists in Docker Compose overrides | medium | Compose **concatenates** the `ports` key across `-f` files instead of replacing. `ports: []` in an override does NOT clear the inherited publish. To replace, use Compose 2.24+ `ports: !reset [...]` or remove the publish from the base file entirely. Don't assume override semantics on this key. |
| Building DATABASE_URLs via string interpolation of secrets | medium | `postgres://user:${POSTGRES_PASSWORD}@host/db` breaks silently when the password contains URL-unsafe characters (`/`, `+`, `=` from base64). Use URL-safe encoding (`openssl rand -hex` instead of `-base64`) or URL-percent-encode the password before interpolation. Zod `.url()` validation catches it at boot but only after the bad password is already committed to the postgres data volume. |

### F1 — cloud-init.yaml has 4 known bugs

`infra/cloud-init.yaml` doesn't produce a working VM on first boot. Live VM is in the correct end state only because of manual fixes. A fresh `terraform taint oci_core_instance.timeline + terraform apply` today reproduces the bugs.

- [ ] **docker-compose-plugin package source.** Add Docker's official apt repo via cloud-init `apt:` config, OR switch to `docker-compose-v2` Ubuntu universe package (verify availability on jammy ARM64 first).
- [ ] **iptables rule ordering.** runcmd uses `-A INPUT` (append) for 80/443 ACCEPTs, landing them after the catch-all REJECT. Change to `-I INPUT <pos>` to insert before the REJECT.
- [ ] **git clone runs unconditionally.** Gate the clone on a success-marker file from the prior package install step, OR detect partial install and bail with a clear error.
- [ ] **nginx cache directory missing.** `ops/nginx/timeline.conf` declares `proxy_cache_path /var/cache/nginx/public_reel`; cloud-init doesn't create the directory. Add a runcmd that does `mkdir -p /var/cache/nginx/public_reel && chown -R www-data:www-data /var/cache/nginx/public_reel` BEFORE nginx is enabled.

**Verification:** After patching, run `terraform taint oci_core_instance.timeline && terraform apply`, SSH in fresh, walk through Phase 8 Wave 3 deploy from scratch with no manual fixes. Then mark this item closed.
