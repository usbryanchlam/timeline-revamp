## Root-Cause Analysis

### Mainthread work breakdown (top 5)

| Group            | Duration |
| ---------------- | -------: |
| Script Evaluation | 3,895 ms |
| Other            | 327 ms   |
| Style & Layout   | 34 ms    |
| Paint/Composite  | 30 ms    |
| GC               | 29 ms    |

Script evaluation dominates the mainthread budget under 4x CPU throttling
(Moto G4 emulation). The largest contributors are React 19 + maplibre-gl —
both ship as parsed/compiled bytecode before LCP can resolve.

### Largest network resources

| Resource                        | Transfer Size |
| ------------------------------- | ------------: |
| `assets/maplibre-wqmL2Hxp.js`   | 283 KB        |
| `assets/index-BrQlRDdc.js`      | 170 KB        |
| `seed-photos/hong-kong/1.jpg`   | 366 KB        |

### LCP element

Lighthouse could not identify a definite LCP element — the public reel
renders a `<MapPoster>` fallback while maplibre loads, then swaps to the
canvas after hydration. Net effect: the LCP moment is dominated by JS
evaluation of the maplibre chunk, not by image/text paint.

## Why this defers to v1.1 (not fixed in 11-02)

The fix surface spans multiple architecture seams and exceeds a one-commit
scope:

1. **Lazy maplibre import** — already done in `Reel.tsx` (React.lazy +
   manualChunks). Yet maplibre is still imported eagerly by `OrbitReel.tsx`
   and `GlobeReel.tsx`, defeating the public-reel split. Fix: gate
   `OrbitReel`/`GlobeReel` behind their respective routes (already true at
   the router level, but the JS bundle still includes them via React's
   module graph).
2. **Hong-Kong seed photo (366 KB)** is the largest network payload. A 75%-
   quality resize to ~150 KB would knock ~500 ms off LCP under throttle. Fix
   requires re-encoding the seed photo set + updating `src/data/seeded-cities.ts`.
3. **React 19 runtime (~70 KB after gzip)** — switching to react-dom/server
   pre-rendering of the LCP poster would also help; out of scope for a v1.0.0
   late polish phase.

These three together would likely lift perf from 40 → 75-80; reaching ≥ 90
requires a fuller perf pass (PRELOAD hints, font-loading strategy review,
LCP poster pre-rendering). v1.1 is the correct horizon for that work.

## v1.0.0 ship rationale

The Lighthouse baseline IS the v1.0.0 launch artifact, regardless of the
score. The script captures perf evidence for future regressions to be
detected against. PERF-04 (CLS) ships green. PERF-01 and PERF-03 ship as
explicit deferrals with diagnosis on file.
