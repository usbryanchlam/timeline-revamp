<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub OIDC issuer → OCI Identity Domain | JWT exchanged for UPST via `/oauth2/v1/token`. Trust gate enforced by Identity Propagation Trust subject mapping. |
| PR identity → OCI tenancy | Read-only via inspect+read object-family policies. Cannot mutate. |
| Main-branch identity → OCI tenancy | Full manage on the compartment. Gated by GitHub Environment `production` manual reviewer. |
| Scheduled cron → OCI tenancy | Inherits main-branch identity for read; only opens GH issues, never mutates OCI. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08.1-05 | Elevation of Privilege | Auto-apply on push without manual gate would let a buggy or malicious commit on main destroy infrastructure | mitigate | `apply` job has `environment: production`; GitHub Environment is configured with `usbryanchlam` as required reviewer (manual UI step in DEPLOY.md Bootstrap section); apply pauses on "Waiting for review" until approved; supplementary `concurrency: { group: terraform-apply, cancel-in-progress: false }` prevents racing applies |
| T-08.1-06 | Spoofing | OIDC subject scope too permissive (e.g., `repo:OWNER/REPO:*`) would let any branch / any workflow request UPST and call OCI | mitigate | D-19 two-identity pattern: trust has exactly 2 `impersonation_service_users` rules with EXACT subject strings — `repo:usbryanchlam/timeline-revamp:ref:refs/heads/main` for write, `repo:usbryanchlam/timeline-revamp:pull_request` for read; no wildcards; acceptance criteria grep-enforces absence of `:*` patterns |
| T-08.1-04-pr-creep | Elevation of Privilege | Compromised PR identity escalates to apply if its policy is over-broad | mitigate | `gha_pr_reader_inspect` policy grants ONLY `inspect all-resources` + `read object-family`; never `manage` or `use`; acceptance criteria grep-enforces (negative pattern); PR identity in `impersonation_service_users` is bound to a different Service User than `gha_deployer` |
| T-08.1-fork | Spoofing | Fork PR submits malicious `infra/terraform/**` change attempting to run with OIDC token | accept | GitHub default security: forks don't get OIDC tokens unless maintainer approves; Pitfall 10 documented in DEPLOY.md; solo-project context per CONTEXT D-19 deferred section |
| T-08.1-cron-secret | Information Disclosure | Drift detection cron runs with same OIDC + state-backend SK env vars; if compromised, attacker learns OCID structure of all managed resources | accept | Plan output in GH Issues exposes OCIDs and resource counts but not secrets; state file (which DOES contain sensitive OCIDs) stays in private bucket; OCIDs alone aren't a privilege grant without the trust |
| T-08.1-upst-replay | Tampering | UPST is reused across plan + apply jobs if both run in same workflow | mitigate | UPST has 60-minute TTL (RESEARCH Assumption A7); plan + apply auth via SEPARATE token-exchange steps in each job (no token persisted across jobs); `~/.oci/upst.token` written per-job and ephemeral with the runner VM |
